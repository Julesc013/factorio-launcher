## Product result

R3.6 uses the frozen R3.5 architecture and adds product behavior rather than
another consolidation:

- precise outcomes survive direct and machine transports;
- UTF-8 path boundaries, typed `ExecuteRunRequest`, and typed Setup evaluation
  replace ambiguous boundary behavior;
- bounded, no-follow Windows, Linux, and macOS discovery providers use explicit
  provenance and target-native roots;
- 58 command contracts, 56 registered routes, 130 schemas, and 101 refusal
  codes include deterministic status, paths, capabilities, onboarding, and
  explanation commands;
- generated C#, Objective-C, and C++ catalogs carry runtime IDs, request fields,
  availability, reason, risk, effects, localization, grammar, and renderers;
- the TUI is a functional direct client with interactive/noninteractive modes,
  cancellation, timeouts, bounded output, and target-specific packages;
- WinForms and AppKit expose generated required and optional non-execution
  workflows through generic forms and bounded cancellable stdio transports;
- the hermetic journey proves discovery through recovery across live CLI,
  direct-client, stdio, TUI, and desktop-client evidence.

AppKit remains compile-only. The daemon remains unavailable. The older
OS-neutral TUI scaffold profile remains unpublished.

25% threshold. A fresh 21-sample comparison passed the same enforced threshold,

deleted or promoted into a cross-platform guarantee.

state break, setup mutation, network access, frontend backend logic, or human
claim promotion. Remaining quarantines are explicit below.
