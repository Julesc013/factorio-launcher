## Architecture result

R3.5 completes the intended consolidation without changing the three-repository
ownership model, durable root grammar, C++ language level, or public v1 ABI
layouts. The mandatory wave now has:

- zero first-party manual-JSON exceptions;
- zero critical-I/O exceptions;
- an allocator-backed Universal Launcher registry instead of the 32-command
  ceiling;
- 51 command contracts and 49 registered first-class routes;
- truthful command availability, capability, effect, risk, refusal, schema,
  ownership, binding, visibility, deprecation, renderer, localization, and CLI
  grammar metadata;
- generated native command IDs, descriptors, lookups, write policy, complete
  shell grammar, frontend catalog, help, and reference documentation;
- stable JSON document views, deterministic builders/writer, richer outcomes,
  validated strong IDs, and typed persistent stores;
- an injected SetupGateway with explicit unavailable and Universal Setup
  implementations;
- exception barriers and preserved v1 layout/symbol proof at C ABI boundaries;
- reusable direct sessions and bounded `rpc --stdio` process transport with
  request IDs, protocol negotiation, output limits, cancellation, timeouts,
  concurrent pipe draining, and process-tree termination;
- explicit Windows, Linux, and macOS randomness, process, user-path, and
  portability sources;
- R3.2, R3.3, and R3.4 compatibility fixtures;
- a resumable AIDE WorkUnit dependency graph with advisory-only validation
  caching and exact green checkpoints.

`utility.operation` and `setup.operation` remain only as deprecated
compatibility-normalization contracts. They are not registered authoritative
backend multiplexers. `portable_tui_x64` remains experimental and unpublished,
and CI does not pretend it is a promoted built artifact.

## Advisory benchmark comparison

The R3.4 Windows snapshot reported startup 14.1258 ms, its then-combined command
measurement 15.3430 ms, archive inspection 0.0596 ms, mod inspection 0.0454 ms,
diagnostic traversal 0.4162 ms, and package hashing 2.5837 ms.

The R3.5 host-identified baseline reports startup 15.8370 ms, ordinary
`product.inspect` dispatch 17.4924 ms, full 49-route graph materialization
34.6644 ms, archive inspection 0.0533 ms, mod inspection 0.0385 ms, diagnostic
traversal 0.3994 ms, and package hashing 2.5880 ms. The graph cost is now
separated from ordinary dispatch, so the old and new command figures are not a
single like-for-like series. A nine-run enforced comparison passed the committed
25% advisory threshold. These microbenchmarks are host-local evidence, not
cross-platform performance guarantees.
