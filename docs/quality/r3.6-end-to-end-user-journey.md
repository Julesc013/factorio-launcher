# R3.6 End-to-End Non-Execution Journey

`FACMAN-END-TO-END-USER-JOURNEY-01` proves the useful product path without a
licensed Factorio binary, network access, setup mutation, or process execution.
The authoritative scenario is
`tests/test_end_to_end_user_journey.py`.

## Scenario

The test creates disposable Unicode workspaces and executes:

```text
empty status -> bounded fixture discovery -> install import and inspect
-> instance create -> local mod import -> modset lock and verify
-> save list and backup -> launch plan and preflight
-> diagnostics export -> instance export and import -> recovery inspect
```

It snapshots every source fixture before the journey and verifies the same
digests afterward. Persistent writes are confined to the temporary source and
target workspaces plus an explicit temporary output directory.

## Transport evidence

| Surface | Evidence | Claim |
| --- | --- | --- |
| Direct FLB client | `facman_client_smoke` executes `workspace.status` | live native direct transport |
| stdio process client | journey RPC and `facman_client_smoke` | live bounded machine transport |
| CLI | complete stateful scenario | full non-execution journey |
| TUI | live `workspace.status` over the completed workspace | functional second frontend |
| WinForms | generated write-command client smoke | runtime client proof, not GUI automation |
| AppKit | generated catalog/transport checks and target compile | compile-only; no bundle runtime claim |
| extracted packages | target package runtime smokes | target-specific CLI/TUI package entrypoints |

The CLI, machine transport, direct client, and TUI compare the canonical
`workspace.status` command identity. Desktop catalog tests prove the same
runtime IDs and generated payload mapping. AppKit remains compile-only until an
actual application-bundle invocation exists.

## Authority boundary

The journey sends `run.execute` through the machine protocol with a valid typed
instance ID and requires `unavailable` / `isolation_not_proven`. No launch
history or execution promotion is part of this proof. H1, H2, and H3 therefore
remain human authority gates.

Security, corruption, link, archive-budget, stale-transaction, setup-disabled,
provider-unavailable, timeout, cancellation, and read-only-package negatives
remain enforced by their focused regression suites and the strict checker.
