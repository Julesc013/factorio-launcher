# R3.5 command benchmark rationale

The pre-R3.5 `command_dispatch` measurement invoked `command-graph inspect`.
After first-class route consolidation, that command serializes 49 registered
descriptors instead of measuring only ordinary dispatch. On the reference
Windows host it rose from 14.8 ms to approximately 34.5 ms, while
`product inspect` remained approximately 17.2 ms against 15.9 ms process
startup.

R3.5 therefore separates the two costs:

- `command_dispatch` invokes `product inspect --json` and measures normal
  client, graph lookup, handler, and response-envelope overhead.
- `command_graph_materialization` invokes `command-graph inspect --json` and
  measures full generated graph serialization.

The baseline is refreshed only after recording this distinction. The higher
graph-materialization number is accepted as the bounded cost of exposing the
expanded first-class registry, not described as a dispatch regression.
