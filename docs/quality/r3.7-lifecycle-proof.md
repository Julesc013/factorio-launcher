# R3.7 lifecycle proof

This checkpoint proves the non-executing instance/content lifecycle. It does
not promote `run.execute`, real Factorio isolation, setup mutation, daemon
publication, credentials, signing, or release/publication authority.

## Main journey

`tests/test_r37_lifecycle_journey.py` executes an empty Unicode workspace
through discovery/import, instance create/inspect/verify/clone, profile
create/plan/apply, local mod index, modset plan/apply/verify, save
index/associate, snapshot create, live-instance-to-snapshot diff, archive,
restore under a new ID, server plan/export, snapshot/save retention plans, and
recovery inspection. Fixture hashes are checked before and after.

The completed workspace is then inspected through:

- direct CLI/FLB dispatch;
- bounded `rpc --stdio`;
- TUI noninteractive direct transport;
- TUI guided forms;
- TUI bounded process transport;
- the headless WinForms generated-client probe in `tests/frontend_harness`.

AppKit consumes the same generated catalog and payload grammar, but native
AppKit runtime proof is not claimed from Windows. Target CLI/TUI package
runtime proof remains part of the WU11 target matrix.

## Performance corpus

The explicit full-scale test is gated by
`FACMAN_RUN_R37_FULL_PERFORMANCE=1` so ordinary affected-test runs stay fast.
The recorded Windows Release run passed in 124.885 seconds and covered:

- 1,000 local mod archives and roughly 6,000 dependency edges;
- successful bounded resolution plus a one-state fail-closed exhaustion;
- 10,000 structurally indexed saves;
- 100 actual deterministic snapshots;
- a 100-snapshot, 10 GiB simulated retention-policy corpus in
  `facman_snapshot_policy_smoke`;
- a 13,000-file large instance clone/diff with only generated manifest/config
  differences.

Direct in-process structured decoding is bounded at 64 MiB and one million
nodes for the 10,000-save result. Bounded process transport remains 16 MiB,
and TUI display remains 1 MiB.
